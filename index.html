Memory

Shared Memory Guide

이 화면은 여러 모델과 여러 협업자가 같은 프로젝트 기록을 어떻게 함께 읽고, 어떤 내용만 공용으로 남기고, 무엇을 다음 사람과 다음 도구로 넘기는지 설명하는 안내판입니다. 저장소 소개가 아니라 공용 기억이 실제로 이어지는 순서를 보여주는 탭으로 읽으면 됩니다.

Shared Memory Guide

무엇이 공용 메모리에 남고 어떤 순서로 전달되는지

위 도식은 설명용 큰 그림입니다. 공용 메모리의 중심이 무엇인지, planner-agent-ko / architect-agent-ko / idea-agent-ko가 남긴 내용이 어디로 모이는지, 그리고 그 결과가 다음 모델과 협업자에게 어떤 순서로 전달되는지를 먼저 읽기 쉽게 보여줍니다.

Mermaid Guide

압축 구조도 읽는 법

아래 Mermaid는 위 설명용 도식을 더 짧고 정확하게 압축한 버전입니다. 선 색과 점선 규칙을 보면 무엇이 기본 공용 경로인지, 무엇이 private/local에서 승격될 때만 들어오는지, 어떤 결과가 adapter export와 협업 packet으로 빠져나가는지 바로 읽을 수 있습니다.

Shared Lanes

공통 규칙 / 공유 결과물 / 협업 전달

공용 메모리는 이 세 줄기로 읽으면 됩니다. 공통 규칙은 출발 순서를 맞추고, 공유 결과물은 현재 상태를 남기고, 협업 전달은 다음 사람과 다음 모델이 바로 이어받을 수 있게 정리합니다.

Handover Flow

사람과 모델이 이어받는 순서

협업 전달은 긴 설명 카드가 아니라 순서가 중요합니다. 누가 기록을 남기고, 무엇이 공용 허브에 승격되고, 어떤 묶음이 다음 사람과 다음 모델로 넘어가는지 단계별로 읽으면 전체 구조가 훨씬 쉬워집니다.

Shared Layers

공유 메모리 레지스트리

이 표는 공용 메모리에 남는 층만 짧게 정리한 요약표입니다. 개인 취향이나 모델 내부 기억은 여기에서 빼고, 다음 사람과 다음 모델이 같이 읽어야 하는 층만 남겼습니다.

Personalization

User Memory & Personalization

이 화면은 내 작업 스타일과 개인 기억이 어떤 식으로 저장되고, 답변 톤과 작업 순서, handover 문서, 다음 모델용 continuity 기록에 어떻게 반영되는지 보여주는 private 탭입니다. Shared Memory에 자동 공개되지 않는 개인 레이어만 모아 읽을 수 있습니다.

Locked Personalization

로그인 후 개인화 영역 열기

이 영역은 개인 브리프, 개인 메모, 작업 방식, continuity 기록처럼 공개하지 않는 내용을 담습니다. 로그인 전에는 템플릿만 보이고, private 카드만 부분 잠금 상태로 유지됩니다.

Google 세션 또는 개발자 마스터키로 잠금 해제할 수 있습니다.

Private Workspace

Private Vault + Model-local Memory + Continuity Playbook

이 영역은 개인 기억이 어디에 저장되고, 어떤 것은 장기 기억으로 남고, 어떤 것은 현재 세션에만 머물며, 어떤 내용이 다음 모델과 도구로 넘어갈 continuity packet이 되는지 보여 줍니다.

Continuity Archive

장기 / 중기 / 단기 기억과 지속 업데이트 기록

여기에는 개인 기억이 어디에 저장되는지, 장기/중기/단기 기억이 어떤 역할을 하는지, 그리고 시간이 지나도 다른 모델과 도구가 같은 작업을 이어받을 수 있도록 어떤 continuity 기준을 계속 쌓는지 정리합니다.

로그인 후 저장하면 이 브라우저에 개인화 메모가 유지됩니다.

Registry

Model + Tool Matrix

모델과 툴은 각각 따로 읽어야 가독성이 좋기 때문에 풀폭 표로 분리했습니다. 지금 표는 실사용자 기준으로 많이 쓰는 축을 먼저 보이게 정렬했고, 모델 표에는 공식 문서 기준 날짜와 knowledge cutoff를 함께 보고, 툴 표에는 설치 방식, 로그인 조건, 메모리 브리지, 공식 링크를 함께 읽습니다.

Models

모델 비교표

컨텍스트, 출력 한도, 접근 방식, 비용/제한, 추천 상황을 한 번에 비교합니다. 위쪽일수록 실사용자 기준으로 자주 거론되는 모델이고, 각 행의 상세는 읽기용 해석이며 옆 아이콘은 공식 문서로 바로 이동합니다. knowledge cutoff는 공식 공개값이 있을 때만 표기합니다.

Tools

툴 비교표

설치 방식, 로그인 여부, 쿼터, 메모리 브리지, 추천 용도를 중심으로 나눠 봅니다. 위쪽일수록 커뮤니티에서 실제로 많이 쓰는 축을 우선 배치했고, 이 표는 이 툴을 왜 쓰는지와 공용 메모리와 어떻게 연결되는지를 읽는 표입니다.

현재 설치 보기: WSL-Linux

Skills

모델별 스킬 구성 + 계층도

루트 코어에서 모델 허브로, 다시 각 스킬 노드로 내려가는 실제 트리 형태로 재구성합니다. 각 스킬은 hover로 미리 보고, 클릭하면 말풍선이 고정되어 역할과 메모리 패턴을 계속 읽을 수 있습니다.
