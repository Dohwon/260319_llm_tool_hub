# LLM Tool Hub V1 Scope

## 목적

V1의 목표는 이 프로젝트를 "개인용 LLM 운영 허브"로 안정화하는 것이다.
지금 단계에서는 기능을 더 늘리기보다, 이미 들어간 기능을 한 화면에서 관리하고 로그인, 개인화, 프롬프트 재사용 흐름이 끊기지 않게 만드는 것이 우선이다.

핵심 사용자 가치는 아래 4가지다.

1. 모델과 툴을 한 번에 비교하고 고를 수 있다.
2. 중앙 메모리와 개인화 정보를 한 곳에서 관리할 수 있다.
3. 자주 쓰는 프롬프트를 Prompt Forge 계열 흐름으로 다시 꺼내 쓸 수 있다.
4. 크롬 익스텐션 확장 전 단계로서 웹 허브가 기준점 역할을 한다.

## V1 제품 정의

V1은 "통합 허브"를 유지한다.
아직 Prompt Forge, extension, memory 관리 축을 분리하지 않는다.
분리보다 중요한 것은 아래 세 가지다.

- 로그인과 세션 안정성
- 중앙 메모리/개인화 흐름의 일관성
- 프롬프트 재사용과 운영용 문서/데이터의 단일 기준 유지

## 탭별 기능

### Home

역할:
- 프로젝트 전체 상태를 요약해서 보여주는 시작 화면
- 레지스트리, 메모리, 추천 상태를 빠르게 훑는 대시보드

V1에서 해야 하는 일:
- 현재 시스템이 어떤 데이터 소스를 기준으로 움직이는지 명확히 보여주기
- 추천 결과와 운영 상태가 한 번에 보이게 유지하기

완료 기준:
- 처음 들어온 사용자가 "이 허브가 무엇을 하는지"를 10초 안에 이해할 수 있다

### Memory

역할:
- `global_memory/memory_profile.json` 중심의 중앙 메모리 현황 확인
- 장기/중기/단기 문서와 연결된 운영 기준점 확인

V1에서 해야 하는 일:
- 중앙 메모리 원본과 파생 문서의 관계를 깨지 않기
- 메모리 변경이 추천과 개인화에 어떤 영향을 주는지 보이게 하기

완료 기준:
- 메모리 관리 기준이 "파일 여러 개를 따로 보는 작업"이 아니라 "허브에서 상태를 파악하는 작업"이 된다

### Personalization

역할:
- 로그인된 사용자의 개인화 메모와 접근 상태를 관리
- Google OAuth 또는 개발자 마스터키를 통해 잠금 해제

V1에서 해야 하는 일:
- 로그인 후 세션 유지가 안정적이어야 한다
- 개인화 메모 저장/복원 흐름이 명확해야 한다
- 로그인 상태에 따라 잠금/해제 UX가 일관돼야 한다

완료 기준:
- 로그인 이슈 없이 개인화 메모를 기준으로 Prompt 계열 기능이 열린다

### Skills

역할:
- 설치된 스킬 레지스트리와 운영 상태를 확인
- 어떤 스킬이 어떤 목적에 맞는지 빠르게 파악

V1에서 해야 하는 일:
- 스킬 목록, 설명, 설치 상태가 혼동 없이 정리되어 있어야 한다
- 내부적으로 어떤 스킬이 프로젝트 운영과 연결되는지 보이게 해야 한다

완료 기준:
- "무슨 스킬이 있는지 모르겠다"가 아니라 "이 상황엔 뭘 쓰는지 안다" 수준까지 정리된다

### Prompt Studio 계열

이 프로젝트의 실질적인 수익/확장 포인트는 Prompt 계열 탭들이다.
지금은 각각 분리 서비스가 아니라 같은 허브 안의 기능 묶음으로 본다.

#### Prompt Tailor

역할:
- 입력한 의도나 문장을 상황에 맞는 프롬프트로 다듬기
- 개인화 정보와 결합된 재작성/개선 흐름 제공

V1에서 해야 하는 일:
- 로그인 상태와 사용량 제한이 정확해야 한다
- 생성 결과를 바로 재사용할 수 있어야 한다

#### Prompt Forge

역할:
- 이미 만들었던 프롬프트를 다시 쓰거나, 새 프롬프트를 운영 단위로 관리하는 핵심 기능
- 장기적으로 크롬 익스텐션과 이어질 기준 허브

V1에서 해야 하는 일:
- 저장/복사/재실행 흐름이 빠르고 단순해야 한다
- "한 번 만든 프롬프트를 다시 꺼내 쓰는" 가치가 분명해야 한다

#### Prompt Translate / Live Prompt

역할:
- 프롬프트 번역 및 실시간 프롬프트 생성/실험
- 운영자가 모델/프롬프트 조합을 빠르게 확인하는 실험 구역

V1에서 해야 하는 일:
- 실험 기능은 넣되, 저장/관리 흐름보다 앞서지 않게 한다
- 운영 툴로서의 실용성을 유지한다

## V1에서 하지 않는 것

- Prompt Forge를 별도 서비스로 완전히 분리
- 중앙 메모리 시스템을 외부 SaaS처럼 다중 사용자화
- 결제/플랜/권한 시스템의 대규모 확장
- 정교한 협업 기능, 팀 워크스페이스, 조직 관리
- 복잡한 AI 자동화 워크플로 빌더

## 기술/운영 원칙

- 정답 데이터는 `data/registry.json`, `data/projects.json`, `global_memory/memory_profile.json`을 기준으로 유지한다
- 공개 UI는 정적 HTML/CSS/JS 우선 구조를 유지한다
- 기능 확장보다 운영 안정성, 로그인 안정성, 문서 기준점 유지가 우선이다
- 중앙 허브 구조는 당분간 유지하고, 분리는 V2 판단 이후 검토한다

## V1 종결 판단

V1은 아래 조건을 만족하면 종결로 본다.

1. Google OAuth와 개발자 로그인 모두 안정적으로 동작한다.
2. 개인화 잠금 해제 후 Prompt 계열 기능 사용 흐름이 끊기지 않는다.
3. 메모리/레지스트리/스킬/프롬프트 재사용 기능이 한 허브 안에서 정리되어 있다.
4. 이 프로젝트를 이후 `Prompt Forge 분리` 또는 `extension 강화`의 기준 베이스로 사용할 수 있다.
